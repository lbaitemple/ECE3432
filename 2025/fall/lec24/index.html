Module 4: Mini Pupper Camera and Line Following

  • Enabling and accessing the Mini Pupper camera
  • Image processing basics with OpenCV
  • Using camera input in ROS2
  • Implementing a basic line-following algorithm

Enable Camera in [mini_pupper_bringup]

  • Change mini_pupper_2.yaml in ros2_ws/src/mini_pupper_ros mini_pupper_bringup/config/mini_pupper_2.yaml
sensors:
  lidar: false
  imu: true
  camera: true
ports: 
  lidar: '/dev/ttyAMA1'

Important

  • check if camera yaml file exists
mkdir -p ~/.ros/camera_info
cp ~/mini_pupper_ros/mmal_service_16.1.yaml ~/.ros/camera_info

Camera

  • after revision, compile in ros2_ws
colcon build --packages-select mini_pupper_driver

Bring up

  • need to launch all hardware

  • drift correction in champ

export ROS_DOMAIN_ID=0
source ~/ros2_ws/install/setup.bash
ros2 launch mini_pupper_bringup bringup.launch.py \
            hardware_connected:=True drift_correction:=0.1

View Image

  • on LCD
  • on Web
export ROS_DOMAIN_ID=0
source ~/ros2_ws/install/setup.bash
ros2 run mini_pupper_driver display_interface --ros-args \
         -r mini_pupper_lcd/image_raw:=/image_raw
  • think about putting it into a ros2 launch file

export ROS_DOMAIN_ID=0

  • Single Viewer: http://localhost:5000 (or your custom port)

Line following

  • line following code in mini_pupper_recognition
  • use another terminal to send recogntion code
# Terminal 2 (ssh or PC)
export ROS_DOMAIN_ID=0
source ~/ros2_ws/install/setup.bash
ros2 launch mini_pupper_recognition recognition.launch.py #pid:=false if want to disable pid
