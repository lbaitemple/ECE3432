Introduction to Windows Subsystem for Linux (WSL)

Windows Subsystem for Linux (WSL) allows you to run a Linux environment directly on Windows.

  1. Enable WSL on Windows
  2. Install Ubuntu 22.04 LTS
  3. Set up your Linux user account (username: ubuntu)
  4. Configure basic system settings

Step 1: Enable WSL Feature

Open PowerShell as Administrator and run:

wsl --install -d Ubuntu-22.04

If this command doesn’t work (older Windows versions), use:

dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

Restart your computer after enabling these features.

  1. Open Microsoft Store
  2. Search for “Ubuntu 22.04 LTS”
  3. Click “Get” to install

Step 3: Create User Account

When prompted, enter:

  • Username: ubuntu

  • Password: (choose a secure password)

Step 4: Update System Packages

Run these commands in your Ubuntu terminal:

sudo apt update && sudo apt upgrade -y
sudo apt install ubuntu-wsl -y

  • Access Windows files from Linux: /mnt/c/

  • Access Linux files from Windows: \\wsl$\Ubuntu-22.04

  • Recommended terminal: Windows Terminal

WSL Basic Commands

wsl --list --verbose   # List installed distributions
wsl --shutdown         # Stop all WSL instances
wsl -d Ubuntu-22.04    # Start specific distribution

  • Create the ROS 2 Workspace

Open a terminal and run the following commands to create the workspace and navigate to the src folder:

mkdir -p ~/ros2_ws/src
cd ~/ros2_ws/src
  • mkdir -p: Creates the ros2_ws/src directory if it doesn’t already exist.
  • cd: Navigates to the source directory where you will clone the packages.
  • This downloads the ros2-dev branch of the Mini Pupper ROS repository into the mini_pupper_ros directory.
git clone https://github.com/mangdangroboticsclub/mini_pupper_ros.git -b ros2-dev mini_pupper_ros

Import Required Packages

The .minipupper.repos file lists additional dependencies. Use vcs to clone them recursively:

vcs import < mini_pupper_ros/.minipupper.repos --recursive
  • Ignore Heavy Simulation Components if needed

To avoid compiling resource-heavy packages like Gazebo and Cartographer on Raspberry Pi:

touch champ/champ/champ_gazebo/AMENT_IGNORE
touch champ/champ/champ_navigation/AMENT_IGNORE
touch mini_pupper_ros/mini_pupper_simulation/AMENT_IGNORE
touch mini_pupper_ros/mini_pupper_navigation/AMENT_IGNORE

These AMENT_IGNORE files prevent colcon from building the specified folders.

  • Install Dependencies by navigating back to the workspace root:
cd ~/ros2_ws

Install ROS package dependencies using rosdep

rosdep install --from-paths src --ignore-src -r -y

or

rosdep install --from-paths src --ignore-src -r -y \
  --skip-keys=joint_state_publisher_gui \
  --skip-keys=rviz2 \
  --skip-keys=gazebo_plugins \
  --skip-keys=velodyne_gazebo_plugins
  • Installs missing system dependencies for all packages in the src folder manually

Install additional ROS 2 packages:

sudo apt install -y ros-humble-teleop-twist-keyboard ros-humble-teleop-twist-joy
sudo apt install -y ros-humble-v4l2-camera ros-humble-image-transport-plugins
sudo apt install -y ros-humble-rqt*

These packages support keyboard/joystick teleoperation, camera input, and visualization tools.

Install Python PID library:

pip3 install simple_pid
  • Build the Workspace with Minimal Load
MAKEFLAGS=-j1 colcon build --executor sequential --symlink-install
  • MAKEFLAGS=-j1: Limits the build to a single core to prevent overheating or crashing.
  • --executor sequential: Builds packages one at a time.
  • --symlink-install: Enables easy modification without rebuilding.

  • ✏️ Example: talker_listener.py
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class Talker(Node):
    def __init__(self):
        super().__init__('talker')
        self.publisher_ = self.create_publisher(String, 'chatter', 10)
        timer_period = 0.5
        self.timer = self.create_timer(timer_period, self.timer_callback)

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello from ROS 2!'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: {msg.data}')

class Listener(Node):
    def __init__(self):
        super().__init__('listener')
        self.subscription = self.create_subscription(
            String,
            'chatter',
            self.listener_callback,
            10)

    def listener_callback(self, msg):
        self.get_logger().info(f'I heard: {msg.data}')

def main(args=None):
    rclpy.init(args=args)
    talker = Talker()
    listener = Listener()
    try:
        rclpy.spin(talker)
        rclpy.spin(listener)
    except KeyboardInterrupt:
        pass
    finally:
        talker.destroy_node()
        listener.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
  • ⚙️ Add Entry Point in setup.py

    'console_scripts': [
        'talker_listener = my_robot_pkg.talker_listener:main',
    ],
},

    1. Publishing to /cmd_vel (Geometry Twist for Robot Movement)
from geometry_msgs.msg import Twist

class VelocityPublisher(Node):
    def __init__(self):
        super().__init__('velocity_publisher')
        self.publisher_ = self.create_publisher(Twist, '/cmd_vel', 10)
        timer_period = 1.0
        self.timer = self.create_timer(timer_period, self.timer_callback)

    def timer_callback(self):
        twist = Twist()
        twist.linear.x = 0.5
        twist.angular.z = 0.3
        self.publisher_.publish(twist)
        self.get_logger().info(f'Published velocity: {twist.linear.x}, {twist.angular.z}')

Run with:

ros2 run my_robot_pkg velocity_publisher

You can monitor it with:

ros2 topic echo /cmd_vel

📡 LiDAR Integration

Subscribe to /scan using sensor_msgs/LaserScan

from sensor_msgs.msg import LaserScan

class LidarSubscriber(Node):
    def __init__(self):
        super().__init__('lidar_subscriber')
        self.subscription = self.create_subscription(
            LaserScan,
            '/scan',
            self.listener_callback,
            10)

    def listener_callback(self, msg):
        self.get_logger().info(f'Min Range: {min(msg.ranges):.2f} m')

Use this to process obstacle distances for navigation or SLAM.
