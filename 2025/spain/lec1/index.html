Linux Outline

Introduction to Linux Commands

  • Linux commands are used to interact with the operating system through the terminal.
  • Commands are typically composed of a command name, options, and arguments.
  • Understanding basic Linux commands is essential for navigating and managing a Linux system.

Navigation

Definition:
Moving through the system and managing files from the command line.

Examples:
- cd, ls, pwd
- cp, mv, rm
- cat, nano, vim

Access

Definition:
Managing who can read, write, or execute files and administer the system.

Examples:
- chmod, chown, chgrp
- adduser, usermod, passwd
- groups, whoami

Control

Definition:
Monitoring and managing system processes and installing software.

Examples:
- ps, top, kill, jobs
- apt, yum, dpkg, rpm
- fg, bg

Automation

Definition:
Using scripts and network tools to automate tasks and monitor system health.

Examples:
- bash script.sh, for, if
- ssh, scp, ping
- df, free, journalctl, /var/log/

Linux File Systems

  • Command: pwd
  • Explanation: Displays the current directory you are working in.
  • Example: ```bash pwd

ls - List Directory Contents

  • Command: ls
  • Explanation: Lists files and directories in the current directory.
  • Options:
    • -l: Detailed list with permissions, owner, size, etc.
    • -a: Includes hidden files.
    • -t: sort files by time.
    • -h: human readable.
  • Example:
  ls -la

cd - Change Directory

  • Command: cd

  • Explanation: Changes the current directory to a specified directory.

  • Usage:

    • cd /path/to/directory: Move to a specific directory.
    • cd ..: Move up one directory level.
    • cd ~: Move to the home directory.

  • Example:

    cd /home/user/Documents

mkdir and rmdir - Create and Remove Directories

  • Commands:

    • mkdir: Creates a new directory.
    • rmdir: Removes an empty directory.

  • Example:

    mkdir new_folder
rmdir new_folder

Why Python

Note

  • Python works on different platforms (Windows, Mac, Linux, Raspberry Pi, etc).
  • Python has a simple syntax similar to the English language.
  • Python has syntax that allows developers to write programs with fewer lines than some other programming languages.
  • Python runs on an interpreter system, meaning that code can be executed as soon as it is written. This means that prototyping can be very quick.
  • Python can be treated in a procedural way, an object-oriented way or a functional way.

Python package

  • A Python package is a collection of modules organized in directories with an __init__.py file.
  • Useful for organizing code for reuse and distribution.
  • standard package
  • customized package
  • find packages at https://pypi.org/
pip install package
pip show package
  1. Create a folder for your package.
  2. Add your Python module files.
  3. Add __init__.py to mark it as a package.
  4. Create setup.py or use pyproject.toml.
  5. Add a README.md and optionally a license file.
  6. Install locally with pip install -e ..

Step 1: Create a Directory

mkdir my_package
cd my_package

Step 2: Create Module Code

Create a subfolder with your logic:

mkdir my_package
touch my_package/__init__.py
touch my_package/core.py

Example: core.py

def greet(name):
    return f"Hello, {name}!"

Step 3: Add setup.py or pyproject.toml

Example setup.py:

from setuptools import setup, find_packages

setup(
    name='my_package',
    version='0.1',
    packages=find_packages(),
    install_requires=[],
)

Step 4: Add a README.md

Include a brief description and usage instructions.

# My Package

A simple example Python package.

Step 5: Optional - Add Tests

Create a tests/ directory:

mkdir tests
touch tests/test_core.py

Example test file:

from my_package.core import greet

def test_greet():
    assert greet("Alice") == "Hello, Alice!"

📦 Build & Install Locally

pip install .

Or:

python3 -m pip install -e .

(Use -e for “editable” install)
