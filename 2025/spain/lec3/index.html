Docker Cross-platform Build

  • WSL (x86)
  • x86 commands
  • Mac (Apple Silicon)

To build Docker images for ARM64 platforms (e.g., Raspberry Pi) on an x86 system using WSL, follow these steps:

✅ Install Required Tools

sudo apt-get update && sudo apt-get install -y --no-install-recommends \
    qemu-user-static binfmt-support docker-compose

✅ Enable QEMU Emulation

sudo update-binfmts --enable qemu-arm
sudo update-binfmts --display qemu-arm

✅ Register QEMU Handlers for All Architectures

docker run --rm --privileged multiarch/qemu-user-static --reset -p yes

This ensures your system can run containers built for multiple architectures using QEMU emulation.

  • run combined commands as
sudo apt-get update && sudo apt-get install -y --no-install-recommends qemu-user-static binfmt-support  docker-compose 
sudo update-binfmts --enable qemu-arm
sudo update-binfmts --display qemu-arm

sudo apt-get install -y qemu-user-static binfmt-support
docker run --rm --privileged multiarch/qemu-user-static --reset -p yes

No emulation is needed when building for ARM64 (same architecture as Raspberry Pi). Just use native Docker on Raspberry Pi.

Mount host files or directories inside a Docker container using the -v option.

📁 Example: Mount a Script

  • /app/myscript.py: file path in the container

Dockerfile Basics

A Dockerfile lets you define an image from code.

📝 Example: Python App Dockerfile

FROM python:3.10-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
CMD ["python", "app.py"]

🔧 Build the Image

docker build -t my-python-app .

Docker Compose: Basic & Advanced

Docker Compose lets you manage multi-container applications using a docker-compose.yml file.

🧰 Basic Example

version: "3.9"
services:
  web:
    image: nginx
    ports:
      - "8080:80"

Run it:

docker-compose up

🛠️ Advanced Options

🔁 Auto-Restart Containers

restart: always

🔒 Limit Resources

deploy:
  resources:
    limits:
      cpus: "0.5"
      memory: 256M

📂 Mount Volumes

volumes:
  - ./data:/data

📦 Build from Dockerfile

build:
  context: .
  dockerfile: Dockerfile

📋 Full Example

version: "3.9"
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "5000:5000"
    volumes:
      - .:/app
    restart: unless-stopped
    environment:
      - ENV=production

🧪 Build and Run

docker-compose build
docker-compose up

🧼 Clean Up

docker-compose down --volumes
